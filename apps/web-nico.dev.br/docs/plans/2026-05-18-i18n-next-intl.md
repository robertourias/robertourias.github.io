# Plano Técnico: Internacionalização com next-intl (PT / EN / ES)

**Spec:** `.ai-core/specs/2026-05-18-i18n-next-intl.md`
**Status:** ready
**Data:** 2026-05-18

---

## Contexto

O app usa Next.js 16.2.1 (App Router) com React 19. Todas as rotas vivem em `apps/web-nico.dev.br/src/app/`. A internacionalização exige mover as rotas para dentro de `app/[locale]/` e adicionar um middleware `next-intl`. É uma refatoração estrutural — todas as tarefas são frontend.

**Arquivos-chave afetados:**
```
src/
  app/
    layout.tsx              → mover para app/[locale]/layout.tsx
    page.tsx                → mover para app/[locale]/page.tsx
    curriculo/page.tsx      → mover para app/[locale]/curriculo/page.tsx
    projetos/page.tsx       → mover para app/[locale]/projetos/page.tsx
    api/                    → NÃO mover (routes de API sem locale)
    components/             → atualizar para consumir translations
    data/curriculum.ts      → adicionar versões EN e ES
    data/chat-context.ts    → refatorar para getSystemPrompt(locale)
  middleware.ts             → criar (novo)
messages/
  pt.json                   → criar (novo)
  en.json                   → criar (novo)
  es.json                   → criar (novo)
i18n/
  request.ts                → criar (config next-intl para RSC)
  routing.ts                → criar (locales e defaultLocale)
```

## Ordem de execução

> As tarefas devem ser executadas em sequência — cada uma depende da anterior.

## Tarefa 1: Instalar e configurar next-intl

**Tipo:** chore
**Agente:** frontend
**Depende de:** nada

Instalar `next-intl` e criar os arquivos de configuração de roteamento e request. Nenhuma página é alterada aqui — apenas a base da lib.

Critérios de aceite:
- [ ] `next-intl` adicionado em `package.json`
- [ ] `src/i18n/routing.ts` criado com `locales: ['pt', 'en', 'es']` e `defaultLocale: 'pt'`
- [ ] `src/i18n/request.ts` criado para integração com RSC (Server Components)
- [ ] `src/middleware.ts` criado com `createMiddleware` do next-intl, aplicando a lógica: `es` → `/es/`, `pt` → `/pt/`, qualquer outro → `/en/`
- [ ] `next.config.ts` (ou `.js`) atualizado com `withNextIntl`
- [ ] Build sem erros: `pnpm build`

Notas:
- Usar next-intl v3+ (compatível com Next.js 16 + React 19)
- O middleware substitui qualquer `middleware.ts` existente
- A lógica de fallback para inglês deve usar `localeDetection` do next-intl ou middleware customizado

**Tipo:** feature
**Agente:** frontend
**Depende de:** Tarefa 3

Criar versões EN e ES de `curriculum.ts`. Estratégia: um arquivo por locale (`curriculum.pt.ts`, `curriculum.en.ts`, `curriculum.es.ts`) com um barrel `curriculum.ts` que exporta `getCurriculumData(locale)`.

Critérios de aceite:
- [ ] `src/app/data/curriculum.pt.ts` — dados atuais (português)
- [ ] `src/app/data/curriculum.en.ts` — versão inglês (summary, bullets de experiência, roles, differentials, skillGroup labels, education degrees)
- [ ] `src/app/data/curriculum.es.ts` — versão espanhol
- [ ] `src/app/data/curriculum.ts` refatorado para exportar `getCurriculumData(locale: string): CurriculumData`

Notas:

## Riscos na execução

1. **next-intl + Next.js 16 + React 19:** verificar compatibilidade de versão antes de instalar. Usar `next-intl@^3.26` ou superior.
2. **`params` como Promise:** No Next.js 15+, `params` em layouts/páginas é uma Promise — usar `await params` ou `use(params)`.
3. **`app/api/` fora do `[locale]/`:** Confirmar que as rotas de API não sofrem impacto do middleware next-intl (configurar `matcher` corretamente).
4. **ChatWidget é Client Component:** Obter o locale via `useLocale()` do next-intl para enviar com a requisição.
