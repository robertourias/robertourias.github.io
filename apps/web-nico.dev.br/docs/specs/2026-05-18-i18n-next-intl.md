# Spec: Internacionalização com next-intl (PT / EN / ES)

**Status:** approved
**Data:** 2026-05-18
**Autor:** planner-agent

---

## Problema

O site nico.dev.br está inteiramente em português, o que limita o alcance para recrutadores e empresas internacionais. Dado que o objetivo do site é apresentar Roberto Nicoletti como candidato sênior, a ausência de versões em inglês e espanhol cria barreira para oportunidades globais.

---

## Cenários de Usuário

- **P1 (crítico):** Como recrutador internacional com browser configurado em inglês, quero acessar o site e ser direcionado automaticamente para a versão em inglês, para avaliar o perfil sem barreira de idioma.
- **P1 (crítico):** Como recrutador latino-americano (ex-ES) com browser em espanhol, quero ser direcionado para a versão em espanhol, para ler o perfil no meu idioma.
- **P2 (importante):** Como qualquer visitante, quero poder trocar o idioma manualmente via seletor na navbar, para ajustar caso a detecção automática não reflita minha preferência.
- **P2 (importante):** Como recrutador internacional, quero que o chat assistant responda com contexto completo sobre Roberto no meu idioma, sem precisar digitar em português.
- **P3 (nice-to-have):** Como usuário, quero que a URL reflita o idioma que estou lendo (ex: `/en/curriculo`) para poder compartilhar um link já no idioma correto.

---

## Requisitos Funcionais

- **FR-001:** O site deve suportar três locales: `pt` (padrão), `en` e `es`.
- **FR-002:** Todas as rotas devem ter prefixo de locale: `/pt/`, `/en/`, `/es/`. A raiz `/` redireciona para o locale detectado.
- **FR-003:** A detecção automática usa o header `Accept-Language` do browser. Regra: se o idioma principal for `es` → redirecionar para `/es/`; se for `pt` → `/pt/`; qualquer outro idioma → `/en/`.
- **FR-004:** Interface completa traduzida: labels, títulos de seção, CTAs, textos da navbar, footer, botões, mensagens de estado (loading, erro, sucesso do formulário).
- **FR-005:** Conteúdo do currículo traduzido nas três línguas: `name`, `title`, `summary`, `experiences[].role`, `experiences[].bullets`, `skillGroups[].label`, `differentials`, `educationList[].degree`. Dados estáticos não-traduzíveis (empresa, período, instituição) permanecem como estão.
- **FR-006:** `chat-context.ts` exporta um `SYSTEM_PROMPT` por locale. O chat usa o prompt correspondente ao locale da página atual.
- **FR-007:** Seletor de idioma na navbar permite troca manual entre PT / EN / ES, preservando a rota atual (ex: `/pt/curriculo` → `/en/curriculo`).
- **FR-008:** Implementação usando `next-intl` com App Router (middleware de detecção + `[locale]` segment).
- **FR-009:** Strings de tradução organizadas em arquivos JSON por locale: `messages/pt.json`, `messages/en.json`, `messages/es.json`.
- **FR-010:** SEO: tag `<html lang="...">` correta por locale e `<link rel="alternate" hreflang="...">` para as três versões em cada página.

---

## Critérios de Sucesso

- [ ] Acessar `nico.dev.br` com browser em inglês redireciona para `/en/` automaticamente.
- [ ] Acessar `nico.dev.br` com browser em japonês (ou qualquer não-PT/ES) redireciona para `/en/`.
- [ ] Acessar `nico.dev.br` com browser em espanhol redireciona para `/es/`.
- [ ] Seletor na navbar troca o locale e preserva a rota atual sem recarregar a página inteira.
- [ ] Página `/en/curriculo` exibe todos os textos em inglês, incluindo bullets de experiência.
- [ ] Chat widget na versão `/en/` usa system prompt em inglês.
- [ ] Tag `<html lang="en">` presente na versão inglês; `lang="es"` na versão espanhol; `lang="pt-BR"` na versão português.
- [ ] `hreflang` alternates presentes em todas as páginas para as três versões.
- [ ] Build sem erros de TypeScript.

---

## Fora do Escopo

- Tradução do conteúdo do blog (feature planejada, ainda não existe).
- Locale automático por geolocalização IP (usar apenas `Accept-Language`).
- Mais de três locales nesta iteração.
- Tradução de subprojetos (cada subprojeto é independente).
- Subdomínios por idioma (`en.nico.dev.br`).

---

## Riscos e Premissas

- **Premissa:** O app usa Next.js App Router — a integração com `next-intl` v3+ via `[locale]` segment é o caminho suportado.
- **Premissa:** Todo o conteúdo atual do site está hardcoded em português nos componentes ou em `curriculum.ts` — nenhum CMS envolvido.
- **Risco:** Mover todas as rotas para dentro de `app/[locale]/` é uma refatoração estrutural significativa que afeta todos os componentes de página. → Mitigação: executar em etapa única com testes de build antes de merge.
- **Risco:** `curriculum.ts` tem tipagem estática — a versão traduzida exigirá tipos por locale ou arquivos separados por locale. → Mitigação: definir no plano técnico a estratégia (arquivo por locale vs. objeto indexado por locale).
- **Risco:** O `chat-context.ts` gera o system prompt dinamicamente a partir de `curriculumData` — com i18n, precisará receber o locale como parâmetro. → Mitigação: refatorar para `getSystemPrompt(locale)`.
- **Risco:** SEO — alterar estrutura de URLs pode causar perda temporária de indexação. → Mitigação: implementar `hreflang` e redirecionar `/` corretamente.
